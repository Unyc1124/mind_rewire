--- Quick notes for interns (where to add stuff)
- Controllers: add new controllers to `app/Http/Controllers/`. If controller is auth-related, follow `app/Http/Controllers/Auth/`.
- Models: add to `app/Models/` (Eloquent models). Create corresponding migration in `database/migrations/` when adding new tables.
- Migrations: create new files in `database/migrations/` (timestamped). Run `php artisan migrate` locally.
- Views: add frontend pages to `resources/views/mainsite/` or `resources/views/admin/` depending on usage. Use existing layout files in `resources/views/layouts/`.
- Routes: add new routes to `routes/web.php` for site routes, `routes/admin.php` for admin routes, and `routes/api.php` for API endpoints.
- Assets (CSS/JS): source files live in `resources/js/` and `resources/sass/`. Compiled/public assets are in `public/assets/`. Use `npm run dev` or `npm run build` (see `package.json`).
- Config: app-wide settings go to `config/*.php`.
- Environment: secrets and environment-specific values go into `.env` (do not commit). `.env.example` shows required keys.
- Tests: add unit/feature tests under `tests/Unit` and `tests/Feature`.

--- Tips for AI or tooling
- The main entry points are `public/index.php` (web) and `artisan` (CLI). Framework is Laravel-style; autoloading follows PSR-4 (check `composer.json`).
- For static assets, modify source files under `resources/` and compile with the toolchain (`npm`, `vite`).
